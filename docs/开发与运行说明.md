# CodeAgent 开发与运行说明

本文档说明如何在本地搭建环境、安装依赖、运行 CLI / Web 服务，以及常见问题处理。面向在本仓库中维护 **codeagent**（人格层）且 **seed / seed-tools** 以独立 Python 包提供的场景。

---

## 1. 架构关系

| 组件 | 职责 |
|------|------|
| **seed** | 核心引擎：LLM 调用、路由、会话、工具运行时、`agent_runtime` 等 |
| **seed-tools** | 内置工具注册与执行 |
| **codeagent**（本仓库） | CLI、HTTP/WebSocket 服务、Web UI 接入、认证与项目路径等 |

运行任意 `codeagent` 命令前，环境中需已安装兼容版本的 `seed` 与 `seed-tools`。

---

## 2. 环境准备

### 2.1 Python 与虚拟环境

建议在仓库根目录使用虚拟环境：

```bash
cd /path/to/codeagent
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
```

### 2.2 安装 seed / seed-tools（可选：纯本地 wheel）

若仅从本地目录安装 wheel（不访问 PyPI）：

```bash
pip install --upgrade pip
pip install --no-index --find-links ~/agent/dist seed seed-tools
```

路径请按你机器上实际 `dist` 目录修改。

### 2.3 可编辑安装 codeagent（含 Web 服务依赖）

在仓库根目录执行：

```bash
pip install -e '.[server]'
```

- 默认依赖：`seed`、`seed-tools`
- `[server]`：Starlette、Uvicorn、APScheduler 等，用于 `codeagent serve`

开发时修改 `codeagent/` 下代码无需重复安装，除非变更 `pyproject.toml` 依赖。

## 3. 常用命令

```bash
codeagent --help
codeagent commands
codeagent tools
codeagent route "你的问题"
codeagent run "你的问题"
codeagent config init          # 写入默认 Markdown 等到配置目录
codeagent serve                # HTTP/Web UI（默认端口 8765）
python -m codeagent --help     # 与上面等价的一种启动方式
```

Web UI 若启用访问令牌：

```bash
codeagent webui-token init     # 首次生成令牌文件
```

LLM 相关能力（如 `chat --llm`、服务端对话）需在配置或环境中设置 API，见下一节。

---

## 4. LLM 与配置文件

- 首次可参考：`codeagent config init` 后的提示，将 `config/seed.env.example` 复制为 `config/seed.env`（路径以 **`project_root()`** 为准，常见为 `~/.codeagent` 或当前工程根下的 `config/`）。
- 环境变量通常支持 `SEED_*` 与 `CODEAGENT_*` 别名，具体以 **seed** 包文档与 `env_config` 为准。
- 未正确配置时，进程仍可启动，但调用 LLM 会失败或返回错误信息。

---

## 5. 本仓库中的兼容性处理（了解即可）

以下逻辑是为适配当前发布的 **seed** wheel 行为；若未来 **seed** 在类上正式提供同名方法，可考虑简化。

1. **`LLMAPIExecutor` 与流式接口**  
   部分 **seed** 构建将 `generate` / `generate_stream` 实现为模块级函数，但未绑定到 `LLMAPIExecutor` 类。  
   **codeagent** 在导入包时会通过 `codeagent.llm.executor` 做一次方法绑定，避免 `run_llm_tool_loop` 中出现 `AttributeError: ... generate_stream`。

2. **`setup_builtin_tools`**  
   **seed-tools** 当前 API 为无参调用并返回 `(registry, executor)`。CLI 中 `tools` 子命令已与该签名对齐。

3. **`/icon.png`**  
   站点图标优先在 **Python 包目录** `codeagent/icon.png` 查找，不存在时再尝试 **仓库根目录** `icon.png`。纯 wheel 安装若未打包图标文件，可能仍需把资源放进包内或通过打包配置带上。

## 6. 常见问题

| 现象 | 处理方向 |
|------|----------|
| `pip install -e '.[server]'` 找不到 seed | 先安装本地 wheel，或配置可用的 PyPI/私有索引 |
| `codeagent serve` 无法加载模块 | 确认已安装 `[server]`：`pip install -e '.[server]'` |
| `/icon.png` 404 | 确认仓库根或 `codeagent/` 下存在 `icon.png`，或检查静态资源是否打进安装包 |
| ASGI 报错与 `generate_stream` 相关 | 确认使用包含兼容性补丁的 **codeagent** 版本，且入口通过 `import codeagent` / `codeagent.llm.executor` 加载后再使用 LLM |
| 后台 `serve` 退出码 143 | 多为进程被 **SIGTERM** 结束（例如手动 kill），不一定是崩溃 |
